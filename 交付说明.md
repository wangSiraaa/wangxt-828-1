# 夜市餐车排位屏 - 交付说明

## 一、项目概述
夜市餐车排位屏 Web 前端系统，采用霓虹夜市风格 UI 设计，支持多角色协同管理餐车排位。

## 二、端口说明
- **开发服务器端口**: 5173（Vite 默认端口）
- **访问地址**: http://localhost:5173
- **构建输出目录**: dist/

## 三、技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **样式**: TailwindCSS（霓虹主题）
- **状态管理**: Zustand
- **路由**: React Router DOM
- **拖拽排序**: @dnd-kit
- **动画**: Framer Motion
- **图标**: Lucide React

## 四、快速启动

### 1. 安装依赖
```bash
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

### 3. 构建生产版本
```bash
npm run build
```

### 4. 运行 Smoke 测试
```bash
./smoke.sh
```

## 五、功能模块与检查步骤

### 模块 1：餐车管理（摊主角色）
**页面路径**: /trucks

**检查步骤**:
1. 点击导航栏「餐车管理」进入页面
2. 点击「新增餐车」按钮，填写信息后保存
3. 验证新餐车出现在列表中
4. 点击编辑按钮修改餐车信息，验证保存成功
5. 使用搜索框输入关键词，验证筛选功能
6. 使用品类下拉筛选，验证按品类过滤
7. 点击删除按钮，验证删除确认和删除成功
8. 点击排序按钮，验证按评分/名称排序

### 模块 2：排位管理（管理员角色）
**页面路径**: /ranking

**检查步骤**:
1. 点击导航栏「排位管理」进入页面
2. 点击「上移」/「下移」按钮调整餐车排位
3. 验证排位号实时更新
4. 添加同品类连续超过 2 个餐车，验证出现警告提示
5. 查看「排位预览」区域，验证：
   - 卫生评分 < 60 分的餐车不显示
   - 有违规记录的餐车自动排在最后
6. 点击「重置排位」按钮，验证恢复默认顺序

### 模块 3：违规记录（巡查员角色）
**页面路径**: /violations

**检查步骤**:
1. 点击导航栏「违规记录」进入页面
2. 点击「记录违规」按钮，选择餐车和违规类型
3. 验证违规记录成功保存
4. 验证对应餐车的违规计数 +1
5. 在排位管理中验证该餐车被置底
6. 点击「处理违规」按钮，验证违规状态变为已处理
7. 使用状态筛选，验证按未处理/已处理过滤

### 模块 4：现场大屏（展示角色）
**页面路径**: /display

**检查步骤**:
1. 点击导航栏「现场大屏」进入页面
2. 验证页面自动隐藏导航栏，全屏展示
3. 验证餐车名单自动向上滚动
4. 点击「暂停」按钮，验证滚动停止
5. 点击「继续」按钮，验证滚动恢复
6. 验证列表中不包含卫生评分 < 60 分的餐车
7. 验证违规餐车排在列表末尾
8. 点击「退出大屏」返回首页

### 模块 5：系统设置
**页面路径**: /settings

**检查步骤**:
1. 点击导航栏「系统设置」进入页面
2. 修改「最低卫生评分」为 70，点击保存
3. 在排位管理中验证 60-69 分的餐车也被过滤
4. 修改「同品类连续最大数」为 3，点击保存
5. 在排位管理中验证连续 3 个同品类才警告
6. 点击「重置设置」，验证恢复默认值
7. 点击「重置所有数据」，验证数据恢复初始状态

## 六、业务规则验证清单

### ✅ 规则 1：卫生评分低于阈值不能上屏
- **阈值**: 默认 60 分（可配置）
- **验证**: 在系统设置中调整阈值，观察大屏和排位预览变化
- **自动化验证**: smoke.sh 脚本第 5 步

### ✅ 规则 2：同品类连续排位超过限制要提示
- **限制**: 默认连续 2 个（可配置）
- **验证**: 在排位管理中排列 3 个以上同品类餐车，观察警告提示

### ✅ 规则 3：违规餐车自动置底
- **验证**: 给某餐车添加违规记录，在排位预览中观察其位置
- **自动化验证**: smoke.sh 脚本第 5 步

## 七、数据存储说明
- **存储方式**: 浏览器 localStorage
- **存储键值**:
  - `food_trucks`: 餐车数据
  - `food_truck_violations`: 违规记录
  - `food_truck_config`: 系统配置
- **数据持久化**: 刷新页面后数据保留
- **清空方式**: 系统设置 → 重置所有数据

## 九、常见问题

### Q: 端口 5173 被占用怎么办？
A: 修改 vite.config.ts 中的 server.port 配置。

### Q: 数据丢失了怎么办？
A: 检查浏览器 localStorage 是否被清除，或使用系统设置中的「重置所有数据」恢复 mock 数据。

### Q: 如何修改霓虹主题颜色？
A: 编辑 tailwind.config.js 中的 theme.extend.colors.neon 配置。
